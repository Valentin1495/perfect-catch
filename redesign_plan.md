# Perfect Catch — 게임 리디자인 설계서

> 기존 구조의 핵심 문제: "피하기/먹기 판단"이 인지 부담을 만들고, 반복 루프가 단조로워 레벨 5 이후 흥미가 끊김.
> 이 문서는 **모든 오브젝트를 먹어야 하는 구조**로 전환하고, 난이도를 이동 패턴 + 순서/조합 조건으로 발생시키는 리디자인안입니다.

---

## 1. 핵심 구조 변경

### Before (기존)
```
규칙: "빨간 것만 피하기"
→ 플레이어 판단: 이건 빨간가? → 피해야 하나 먹어도 되나?
→ 문제: 두 번 생각해야 함. "나머지는 어떻게 해?"가 항상 모호.
```

### After (리디자인)
```
모든 오브젝트 = 먹어야 함 (기본값)
난이도 = 오브젝트의 이동 패턴 난이도 + 먹는 순서/조합 조건
→ 플레이어 판단: 어떤 순서로? 어느 타이밍에? → 명확한 단일 질문
```

## 2. 새 난이도 축 설계

### 축 1 — 이동 패턴 (피지컬 난이도)

| 패턴 이름 | 설명 | 등장 레벨 |
|-----------|------|-----------|
| 직선 낙하 | 수직으로 떨어짐 | Lv 1~ |
| 좌우 진동 | sin 곡선으로 흔들리며 낙하 | Lv 3~ |
| 가속 낙하 | 처음엔 느리다가 빠르게 | Lv 5~ |
| 지그재그 | 일정 간격으로 방향 전환 | Lv 7~ |
| 추적 | 플레이어 x좌표를 향해 천천히 이동 | Lv 9~ |
| 분열 | 먹으면 2개로 쪼개짐 | Lv 12~ |
| 페이크 | 방향을 한 번 속임 (역방향 튕김) | Lv 14~ |
| 군집 | 3~4개가 묶음으로 동시 낙하 | Lv 16~ |

### 축 2 — 수집 조건 (인지 난이도)

| 조건 이름 | 설명 | 등장 레벨 |
|-----------|------|-----------|
| 자유 수집 | 순서 무관, 전부 먹으면 됨 | Lv 1~4 |
| 색 순서 | 빨강→파랑→노랑 순서로 먹어야 점수 | Lv 5~ |
| 연속 조건 | 같은 색을 3개 연속으로 먹기 | Lv 8~ |
| 교차 조건 | 색이 번갈아 나와야 함 (빨-파-빨-파) | Lv 11~ |
| 카운트 제한 | 한 색을 최대 N개까지만 먹을 수 있음 | Lv 13~ |
| 타임 윈도우 | 특정 색은 등장 후 1.5초 안에 먹어야 함 | Lv 15~ |
| 콤보 유지 | 같은 조건으로 5회 연속 성공해야 점수 배율 | Lv 17~ |

## 4. UI 변경 사항

### 상단 규칙 바 → "현재 조건 바"로 교체
```
기존: "빨간 것만 피하기"  ← 모호함
신규: "빨강 → 파랑 순서로 먹기" + 시퀀스 진행 표시
      [●빨] → [○파] → [○노]   ← 현재 단계 강조
```

### 조건 바 디자인 원칙
- 텍스트 최소화, 시각 최대화
- 현재 먹어야 할 오브젝트를 항상 강조 표시
- 남은 카운트, 타이머 등 조건 진행 상태를 실시간 표시
- 조건 달성 시 즉각 파티클 + 진동 피드백

### 오브젝트 시각 피드백 추가
- 현재 조건상 "다음에 먹어야 할" 오브젝트: 외곽선 펄스 애니메이션
- 타임 윈도우 조건 적용 중: 오브젝트 주변에 줄어드는 타이머 링
- 분열 직전: 오브젝트가 살짝 진동

## 6. 변경 전후 비교

| 항목 | 기존 | 리디자인 |
|------|------|----------|
| 핵심 질문 | "이걸 피해야 해? 먹어야 해?" | "이걸 언제 어떻게 먹지?" |
| 실패 원인 | 잘못된 오브젝트 접촉 | 오브젝트를 놓침 (화면 밖 이탈) |
| 난이도 발생 | 규칙 복잡도 (피하기/먹기 혼합) | 이동 패턴 + 수집 조건 조합 |
| 플레이어 행동 | 이동 (피하거나 접촉) | 이동 + 타이밍 + 순서 판단 |
| 인지 부담 | 높음 (두 번 판단) | 낮음 (단일 판단: 언제/어떤 순서로) |
| 흥미 지속 | 레벨 5 이후 단조로움 | 패턴 조합으로 매 레벨 새로운 경험 |
| 스킬 요소 | 없음 | 슬로우 스킬, 페버 타임 |
| 단기 목표 | 없음 | 미니 미션 시스템 |
